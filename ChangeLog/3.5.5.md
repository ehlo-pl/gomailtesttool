# 3.5.5 — 2026-07-08

## Fixed
- **`msgraph searchandexport` / `msgraph exportmessages` no longer panic on messages with a null subject or id.** The export paths dereferenced `GetSubject()`/`GetId()` unconditionally, unlike every other call site in the file; both fields are optional in Graph, so a matched message with either unset crashed the export mid-run. A new `derefOr` helper supplies `"(no subject)"` / `""` fallbacks instead.
- **`msgraph exportmessages` CSV rows are now consistently 6 columns.** The success row had 6 columns but the error and no-messages rows had 5, producing ragged CSV output.

## Changed
- Removed the duplicated `maskUsername` helpers in the IMAP and POP3 packages; their `exportmessages` handlers now use the shared `security.MaskUsername`, matching the other protocols.

## Documentation
- `docs/protocols/msgraph.md`: documented the previously missing CSV schemas for `getschedule`, `exportinbox`, `searchandexport`, and `exportmessages`.

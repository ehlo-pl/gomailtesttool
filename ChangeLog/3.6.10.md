# Version 3.6.10

## Changes

### Added

- New `--template <file>` / `--template-vars key=value` flags on every
  `sendmail` action (SMTP, gmail, msgraph, EWS, JMAP). The file is processed
  through Go `text/template` (`--template-vars` values are referenced as
  `{{.key}}`; referencing an unsupplied variable is an error), and the file
  extension picks the mode:
  - **`.eml` (full message mode)** — the rendered file is a complete RFC 822
    message. SMTP injects it verbatim into `DATA` (envelope from
    `--from`/`--to`/`--cc`/`--bcc`, falling back to the EML headers); gmail
    sends it verbatim as `Message.Raw`. msgraph, EWS, and JMAP parse the
    rendered EML and map its recognised fields (recipients, subject, text and
    HTML bodies) onto their native send APIs, logging Bcc (EWS) and unmapped
    headers as skipped.
  - **any other extension (HTML body mode)** — the rendered content becomes
    the HTML body, extending the previous gmail/msgraph-only `--body-template`
    (which remains, without variable substitution) to all sendmail protocols.
  - `--template` is mutually exclusive with `--body`, `--bodyhtml`, and
    (gmail/msgraph) `--body-template`; for raw-injected `.eml` templates the
    flags whose content the file already carries (attachments, custom
    headers, priority — and for gmail recipients and subject) are rejected.
  - Environment variables `*TEMPLATE` / `*TEMPLATEVARS` per protocol prefix;
    config-file keys `template` / `template-vars` (list).
- New shared `internal/common/template` package: `key=value` variable parsing,
  `text/template` rendering (`missingkey=error`), and EML parsing
  (`net/mail` + MIME multipart walk with base64/quoted-printable decoding).

### Documentation

- Documented `--template`/`--template-vars` in
  `docs/protocols/{smtp,gmail,msgraph,ews,jmap}.md` and `docs/config-file.md`;
  added previously missing `sendmail` sections to the EWS and JMAP docs.

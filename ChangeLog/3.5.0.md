# 3.5.0 — 2026-07-07

## Added
- **MCP (Model Context Protocol) support in `serve` mode.** The same sendmail capabilities exposed over HTTP REST are now also available as MCP tools (`smtp_sendmail`, `msgraph_sendmail`, `list_backends`), reachable over two transports from one server:
  - **Streamable HTTP** mounted at `POST /mcp` on the existing HTTP server, behind the same `X-API-Key` middleware. Enabled by default; disable with `--mcp=false` (`SERVE_MCP`).
  - **stdio**, via `serve --mcp-stdio` (`SERVE_MCP_STDIO`), for local AI clients (e.g. Claude Desktop/Code) that launch the binary as a subprocess. No HTTP server is started and no API key is required in this mode.
- MCP tools reuse the same validation and send logic as the REST handlers (extracted into a transport-agnostic core), including the Microsoft Graph no-attachments restriction. See `migration/3.5.MCP.md` and `docs/protocols/serve.md`.

## Changed
- The `serve` command's HTTP handlers were refactored to share a transport-agnostic send core (`sendSMTP` / `sendMsgraph`); existing REST behavior and responses are unchanged.
- Adds dependency `github.com/modelcontextprotocol/go-sdk` (v1.6.1), which requires Go 1.25 (the `go` directive in `go.mod` was bumped `1.24.0 → 1.25.0`).

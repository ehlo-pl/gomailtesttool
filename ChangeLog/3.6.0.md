# 3.6.0 — 2026-07-14

## Documentation
- **`.github/workflows/build.yml`: the release-notes "Quick Start Examples" now use the current `--` flag syntax.** These examples are published verbatim into every GitHub Release, but still showed the legacy single-dash style (`-host`, `-port`, `-username`, …). All SMTP/IMAP/POP3/JMAP examples were converted to double-dash flags, and the Microsoft Graph example was restructured from `-action getinbox` to the `getinbox` subcommand (`gomailtest msgraph getinbox --tenantid … --mailbox …`), matching `README.md` and `docs/protocols/`.

## Planned
> Tracked for this release but **not yet implemented** — see `migration/3.6_google.md` for the full design.

- **New `gmail` provider for Google Workspace / Gmail.** A first-class `gomailtest gmail <action>` provider that reaches Gmail/Workspace the enterprise way, mirroring `internal/protocols/msgraph/` and reusing the shared `internal/common/*` machinery:
  - **Auth:** service account with domain-wide delegation (primary, impersonating a Workspace user), a pre-obtained bearer token, or an interactive OAuth loopback flow.
  - **Mail actions:** `sendmail`, `getinbox`, `exportmessages`; **calendar actions:** `getevents`, `sendinvite`; plus `testauth` and `exportbearertoken`. Command/env prefix `gmail` / `GMAIL*`.
- **Shared MIME builder (`internal/common/mime`).** The MIME assembly currently in `internal/protocols/smtp/sendmail.go` is extracted into a reusable package (behavior-preserving), with SMTP migrated onto it so the new `gmail` provider and SMTP share one implementation.

# 3.5.1 — 2026-07-07

## Changed
- **`msgraph searchandexport` / `msgraph exportmessages` now handle Graph eventual consistency.** A successful-but-empty query result is retried with exponential backoff (governed by the existing `--maxretries` / `--retrydelay` flags), because a just-sent message may not be indexed yet. "No messages found" is reported only after all retries are exhausted (~14 s with defaults); `--maxretries 0` restores single-attempt behavior. The not-found outcome itself is unchanged (same output, CSV row, and exit code 0). Other actions (`getinbox`, `exportinbox`, `getevents`, ...) are unaffected — an empty listing remains a final answer. No `ConsistencyLevel` header is involved: all calls are mailbox-scoped, not directory queries.

## Documentation
- `docs/protocols/msgraph.md`: documented the empty-result retry under `searchandexport`, `exportmessages`, and Retry Configuration.
- `INTEGRATION_TESTS.md`: new "Testing Graph Eventual Consistency" section (positive send-then-search path, exhaustion path, and running the unit tests with the race detector on Windows via a WinLibs gcc toolchain).

# Version 3.6.3

## Changes

### Added

- New `gmail` provider for Google Workspace / Gmail, structured like the existing
  `msgraph` provider. Actions: `sendmail`, `getinbox`, `exportmessages`,
  `getevents`, `sendinvite`, `testauth`, and `exportbearertoken`. Authentication
  supports a service account with domain-wide delegation (impersonating
  `--mailbox`), a pre-obtained bearer token, or an interactive OAuth loopback
  flow with an on-disk token cache. Scopes default per-action (least privilege)
  and can be overridden with `--scope`. All `GMAIL*` environment variables are
  supported.
- Shared `internal/common/mime` package that assembles RFC 5322 / MIME multipart
  messages (`Build`), reused by both SMTP and Gmail. It exposes an
  `IncludeBccHeader` toggle so SMTP keeps Bcc in the envelope while Gmail (which
  derives recipients from the headers) includes it.
- New docs page `docs/protocols/gmail.md` with the GCP setup runbook
  (service account, domain-wide delegation, required scopes) and worked examples.

### Changed

- SMTP message assembly (`buildEmailMessage`/`buildMIMEMessage`) now delegates to
  the shared `internal/common/mime` builder; behavior is preserved and guarded by
  the existing SMTP MIME tests.

### Security

- The `gmail exportmessages` search validates `--messageid` and `--subject`
  before building the `rfc822msgid:`/`subject:` query, blocking search-operator
  injection (mirrors the msgraph OData guards).
